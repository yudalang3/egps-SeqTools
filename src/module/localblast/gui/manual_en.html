2. Overview

Task to resolve

A convenient blast software GUI operation module.

Quick start

Any actions are straightforward. Please follow the GUI instructions.

Note: user need to download the blast software first, and provides the bin directory.

  1. Input the necessary parameters on the graphical interface. For the parameters of the path class, the user can quickly complete it by dragging the mouse or copying the path.
  2. Click the Execute button to see the results.

Caution

  1. To avoid any issues, all file paths should be in English.
  2. You may notice that in BLAST XML files, sometimes BLAST replaces your real sequence IDs with its own generated IDs. For example, the query ID becomes 'QUERY_1', 'QUERY_2', and so on. And hit IDs sometimes become 'GNL|BL_ORD_ID| 1', 'GNL|BL_ORD_ID| 2', and so on. In these cases, BLAST will gather the real sequence IDs and their descriptions together.
    So do not be alarmed if you see some strange characters.

Important Parameter Q&A

The output format of BLAST can be controlled by outfmt, with 18 formats available. The commonly used outfmt 6 has a default output of: qseqid sseqid pident length mismatch gapopen qstart qend sstart send evalue bitscore. Additionally, formats 6, 7, 10, and 17 allow for adjustments in the output content.

The information corresponding to these 12 columns is as follows:

  1. Query id: Identifier for the query sequence
  2. Subject id: Identifier for the aligned target sequence
  3. % identity: The percentage of sequence alignment identity
  4. Alignment length: The length of the alignment region that meets the criteria
  5. Mismatches: The number of mismatches in the alignment region
  6. Gap openings: The number of gaps in the alignment region
  7. Q. start: The starting position of the alignment region on the query sequence (Query id)
  8. Q. end: The ending position of the alignment region on the query sequence (Query id)
  9. S. start: The starting position of the alignment region on the target sequence (Subject id)
  10. S. end: The ending position of the alignment region on the target sequence (Subject id)
  11. E-value: The expected value of the alignment results, which indicates how often you would expect to see a match of equal or higher score if the sequences were shuffled at random. The lower the value, the more conserved the function; a higher E-value suggests that the high score is due to GC content or repetitive sequences. This is a very meaningful parameter for judging homology.
  12. Bit score: The bit score value of the alignment results

-outfmt supports the generation of SAM format.
Alignment view options:

  • 0 = Pairwise,
  • 1 = Query-anchored showing identities,
  • 2 = Query-anchored no identities,
  • 3 = Flat query-anchored showing identities,
  • 4 = Flat query-anchored no identities,
  • 5 = BLAST XML,
  • 6 = Tabular,
  • 7 = Tabular with comment lines,
  • 8 = Seqalign (Text ASN.1),
  • 9 = Seqalign (Binary ASN.1),
  • 10 = Comma-separated values,
  • 11 = BLAST archive (ASN.1),
  • 12 = Seqalign (JSON),
  • 13 = Multiple-file BLAST JSON,
  • 14 = Multiple-file BLAST XML2,
  • 15 = Single-file BLAST JSON,
  • 16 = Single-file BLAST XML2,
  • 17 = Sequence Alignment/Map (SAM),
  • 18 = Organism Report

3. Inputs & Outputs (General)

Item Description
Inputs Typically text/table/sequence files (FASTA/TSV/CSV/GFF3, etc.) or URL/directory paths. Follow the GUI parameter hints.
Outputs Typically result files (TSV/FASTA/PNG/PDF, etc.) or console text. Using a dedicated output directory is recommended.
Encoding UTF-8 is recommended. If you see garbled text, check file encoding and system locale.

4. Parameter & Run Tips

  • Paths with spaces: prefer directories without spaces; quote paths if the module supports it.
  • Large files: validate with a small sample first, then run the full dataset.
  • Result check: verify row/entry counts and key fields to ensure the run is correct.

Symptom Possible Cause Suggestion
